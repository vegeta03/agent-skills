# Frontend Design

Build real, production-ready frontend code with a clear aesthetic point of view.

## Activation Triggers

Activate for requests such as:
- build or redesign a component/page/app UI
- improve visual design quality
- create polished frontend interactions

## Planner behavior

Before coding, define:
- purpose and audience
- aesthetic direction (one clear style, not a blend of defaults)
- constraints (framework, performance, accessibility)
- differentiation (what makes this design memorable)
- implementation depth needed for the chosen aesthetic

## Coding behavior

Implement code that is:
- functional and production-grade
- visually cohesive
- intentional in typography, spacing, color, motion, and composition

Design quality rules:
- choose characterful fonts, avoid generic defaults like Arial/Inter/Roboto/system stacks
- use a deliberate palette with clear hierarchy
- apply motion where it adds meaning (staggered reveal, hover, scroll-triggered moments)
- use composition choices that fit the concept (restraint or maximalism)
- use atmospheric background/detail treatment when it strengthens concept (texture, mesh, layered transparency, shadow, grain)

## Anti-patterns

Avoid:
- cookie-cutter layouts
- predictable gradient-heavy defaults (especially purple-on-white cliché combinations)
- repetitive AI-style aesthetics across different requests
- converging repeatedly on the same trendy defaults across generations

## Guardrails

- Match implementation complexity to the chosen aesthetic.
- Preserve accessibility and usability while pushing visual quality.
- Keep output specific to the product context, not generic showcase UI.
